## 通用决策（从 Taibai 泛化）

## D-001: 错误处理策略
- **日期:** 项目初始化
- **上下文:** 源语言的错误处理模式需要统一翻译策略
- **选项:**
  1. 全部用 anyhow — 简单但丢失类型信息
  2. 全部用 thiserror — 类型安全但定义多
  3. 混合策略 — 公开 API 用 thiserror，内部用 anyhow
- **决策:** 混合策略
- **理由:** 公开 API 需要稳定错误类型给调用者处理；内部错误只需传播，无需精确类型
- **后果:** 需在模块设计时明确公开/内部边界

## D-002: 序列化兼容性
- **日期:** 项目初始化
- **上下文:** 翻译后的系统需要与原系统数据格式兼容
- **决策:** 所有跨系统数据结构必须保持 JSON 序列化兼容
- **后果:** serde 属性需精确配置（rename、default、skip_serializing_if）

## D-003: 并发模型选择
- **日期:** 项目初始化
- **上下文:** 源语言并发模型 → Rust 并发模型的映射策略
- **选项:**
  1. 全 tokio async — 统一但可能过度
  2. std::thread — 简单但缺少异步支持
  3. 按场景选择 — 灵活但需逐个决策
- **决策:** 按场景选择（IO 密集 → tokio；CPU 密集/长运行 → std::thread）
- **理由:** 不同场景有不同最优解，一刀切会引入不必要复杂性
- **后果:** 每个并发模块需单独评估

## D-004: 依赖注入模式
- **日期:** 项目初始化
- **上下文:** 源项目可能使用接口/反射做 DI，Rust 需要不同方案
- **决策:** 使用 trait object + 构造器注入
- **后果:** 每个 DI 接口必须同时有 Mock 和 Real 实现

## D-005: 占位符策略
- **日期:** 项目初始化
- **上下文:** 翻译过程中不可避免有暂未完成的部分
- **决策:** 使用 `todo!("PLACEHOLDER: 原因")` 标记，一个 Wave 内必须消除
- **后果:** 占位符不允许跨 Wave 存在

## D-006: unsafe 使用策略
- **日期:** 项目初始化
- **上下文:** C→Rust 翻译中不可避免使用 unsafe
- **决策:** unsafe < 5% 预算；每个 unsafe 块需 Safety 文档
- **后果:** 超预算时暂停新功能，优先消除 unsafe
